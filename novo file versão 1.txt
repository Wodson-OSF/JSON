 versão de teste feito do git na pasta de respositórios. Nesse sentindo, foram feitos os seguintes passos: criou-se uma pasta de repositório, como nome JSON e em seguida feitos alguns comandos para mandar esse teste em txt para o git. Para isso, iniciou o cód (git clone), clonando o repositório remoto. na sequência foi tirada a pasta que estava oculta, como repositório git. Assim, novamente foi aberto o git dentro MAIN. Retornando para a pasta, foi criado novo arquivo em txt. Voltando para o git, digite: GIT STATUS,  para saber como esta seu repositório. E para adicionar no github, precisa dar o comanda git add (sapaço e ponto(GIT ADD .)) e retorno com o comando git status para saber se o arquivo esta pronto para ir para o repositório do github e ser feito o 'commit'. Para isso, coloque o comando git commit -m (se quiser colocar uma msg, pode colocar entre ""). Ele vai pegar todos os repositórios na área de preparação. Não precisa colocar o nome do arquivo. E o arquvivo será inserido. Caso precise saber quais os commits foram feitos só dar o comando git log e ele mostra um histórico dos commits. E para finalizar, mande o repositório para o Github com o seguinte comando no git: git push origin main (empurrar), mandando os arquivos para a nuvem.
